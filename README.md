# 基于新词发现的短信分类前端系统

这是一个自包含的浏览器端演示系统，用于展示“基于新词发现的短信分类方法”的系统界面和核心处理闭环。

## 打开方式

推荐启动本地后端，以支持人工修正分类落盘：

```bash
node server.js
```

然后访问：

```text
http://localhost:3000
```

也可以直接用浏览器打开 `index.html` 查看静态演示，但人工修正只能在当前页面临时生效。

## 已实现界面

- 短信分类输入：支持单条短信输入、参数配置、运行分类与批量上传入口，左侧输入栏在桌面端随页面滚动固定展示。
- 分类总览：展示短信类别分布、样本列表、单条样本分类结果、置信度、新词检测、变异检测和人工修正入口。
- 人工修正分类：样本分类结果支持人工修正，启动本地后端后可写入 `data/corrections.json`。
- 新词发现：基于固定新词词典展示候选词、文本特征、贡献强度、边界稳定、分类相关、置信度、是否属于现有词表和词表状态。
- 变异词：默认展示最近一次分类任务中新发现的 `逾qi`、`催sh`、`免fei`，并自动加入变异词词表；支持展开完整金融类变异词词表、人工调整和确认踢除。
- 新词动态词典：按统一的新词词典展示主词典和观察区，词和置信度与新词发现、任务查看、监督训练保持一致。
- 任务查看：展示单条短信分类、批量短信分类、手动词典更新等任务记录，按时间从近到远排列。
- 监督训练：展示训练时间轴和分类结果，训练词条沿用统一的新词词典与置信度。

## 内置词典口径

新词相关模块共用同一份词典数据：

- 主词典：授信申请 0.89、授信失败 0.91、已获批 0.88、最高额度 0.95、放款失败 0.82、支付宝 0.99、花呗 0.92、已逾期 0.87、提额包 0.85
- 观察区：很抱歉 0.72、综合评分 0.75、授信获批 0.67、本次授信失败 0.65、到账金额 0.63、扣款成功 0.74、逾期金额 0.72
- 现有词表标记：已逾期、花呗、综合评分

## 说明

页面使用前端规则模拟短信分类、新词发现、变异词识别和动态词典闭环。`server.js` 提供无依赖 Node 后端，用于保存人工修正分类，数据写入 `data/corrections.json`。真实模型接入时，可将 `app.js` 中的 `analyze()` 替换为后端接口调用，并保留当前渲染结构。
